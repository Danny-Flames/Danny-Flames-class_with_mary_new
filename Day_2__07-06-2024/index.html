Github and git

Github is a tool used for collaboration among software developers.

You can create, store, modify and perform any other operations on github.


git init

This command initialises git in your folder. In other words, it immediately make that folder to become a local repository.

Running this command will immediately create a '.git' folder in that folder.
What this folder does is that it helps you keep track of all your files and the changes made to it.


git status

This command gives us information of all files or folders that have some changes and git is yet to be aware of those changes.


git add file_name

This helps you add a particular file to git.
This makes git to be aware of that file and folder.


git add .

This command helps you add all the files including those files inside a folder to git.


git commit -m 'your commit message'

This command commits all your ADDED FILES AND FOLDERS to git.


git remote -v

This command tells us the remote(online) repository our local repo is connected to.
It means you asking git to tell you the remote(online) version this local repo is connected to.


git remote add origin url

This command connects your local repository(working folder) to a remote(online) repo.

git push

This command pushes all your work(files and folders) in your local repo to an remote(online) repo.

git pull

This command pull(fetches) all your work in your remote(online) repo into your local repo

git clone url

This command clones(downloads) the online version of the repo into your local machine.

git checkout -b 'branch_name'

This command creates a new branch and checks out to that branch immediately.

git pull origin branch_name

This command pull from a particular branch.

If you're in a branch named 'example_1' and you run 'git pull origin example_2',
What this means is that git will pull all latest changes from example_2 online into your example_1 branch locally.
